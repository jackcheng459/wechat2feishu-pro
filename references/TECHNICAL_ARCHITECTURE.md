# 🏗️ WeChat2Feishu 技术架构与原理手册

本项目是一套完整的“微信公众号文章采集 → 本地知识库存档 → 飞书云端分发”的自动化流水线。本文档旨在详细拆解其底层逻辑与各组件间的协同原理。

---

## 🛠️ 一、 核心架构：三层协同模型

WeChat2Feishu 的运作基于三层逻辑的紧密耦合：

### 1. 采集引擎层 (Core Python Engine)
这是整个项目的“心脏”，运行在您的本地计算机。
- **采集技术**：采用 **Playwright**（自动化浏览器渲染）。它不只是抓取 HTML 源码，而是真正模拟浏览器加载，能触发微信图片的懒加载逻辑。
- **图片攻坚**：利用网络底层拦截（Request Interception）技术，在浏览器渲染图片的瞬间捕获其 **Binary（二进制）原数据**。这样可以绕过微信 CDN 的所有防盗链校验。
- **格式处理**：将复杂的 HTML 清洗、规整，转化为标准的 Markdown 格式。

### 2. 云端通信层 (Feishu Open API)
这是连接本地数据与飞书云端的“高速公路”。
- **身份认证**：通过 OAuth 2.0 机制获取 **User Access Token**（代表您本人）或 **Tenant Access Token**（代表机器人应用）。
- **原生转换**：利用飞书 `import_tasks` 接口。我们不手动拼装复杂的飞书文档块，而是将本地生成的 MD 文本发送给飞书官方引擎进行转换，确保排版 100% 官方标准。
- **智能注入**：转换完成后，通过 `Patch` 接口回填图片，并利用 `Pillow` 计算比例，确保图片不拉伸。

### 3. 指令调度层 (OpenClaw / Bot Interface)
这是整套系统的“大脑”与“遥控器”。
- **自然语言解析**：OpenClaw 接入飞书机器人，理解用户的意图（例如发送了一个 URL）。
- **技能映射 (Skill Mapping)**：OpenClaw 通过读取 `.openclaw/skills/wechat2feishu.md` 配置文件，知道如何调用本地的 `main.py` 脚本。
- **闭环交互**：OpenClaw 负责在对话框里展示预览、列出文件夹菜单、并最终反馈成功链接。

## 🔄 二、 完整工作流 (Workflow)

当你在飞书里发给机器人一个链接时，背后发生了什么？

1.  **触发**：OpenClaw 捕捉到 `mp.weixin.qq.com` 域名，激活 `wechat2feishu` 技能。
2.  **抓取 (Scrape)**：OpenClaw 远程启动本地 Python，调起 Playwright 开启无头浏览器抓取。
3.  **解析 (Process)**：脚本提取标题、作者、正文，并将图片转为本地 Base64 缓存。
4.  **询问**：OpenClaw 调用 `list-folders` 接口获取您的飞书目录，在对话框里发给您选。
5.  **转存 (Save)**：
    -   脚本将 MD 上传给飞书进行官方转换。
    -   生成的文档自动移动到指定位置（如 Wiki 节点）。
    -   图片二进制数据被注入到文档对应的位置。
6.  **备份**：脚本在本地 `exports/` 目录下同步生成一份图文并茂的文件夹备份。

## 🎨 三、 关键技术特性

- **反爬策略**：动态 IP 模拟、User-Agent 伪装、底层响应拦截。
- **高保真还原**：
    -   **图片比例**：通过 `Pillow` 获取 `(width, height)`，强制 Patch 比例。
    -   **格式兼容**：解决了微信 MD 转换中常见的“标题粘连”和“代码块截断”问题。
- **双轨存档**：云端（飞书）便于共享协作，本地（MD文件）便于长期检索。

## 🛡️ 四、 安全说明

- **密钥管理**：所有的 `APP_ID` 和 `APP_SECRET` 均存储在本地 `.env` 文件中，绝不上传至 GitHub。
- **授权机制**：Token 自动刷新，支持 Tenant 模式实现无人值守。
